# Business Listings Dashboard

A full-stack dashboard that collects, stores, and visualizes 600+ Indian business listings.

## Tech Stack
- **Frontend**: React.js + Recharts (Bar & Pie charts)
- **Backend**: FastAPI (Python)
- **Database**: MySQL 
- **Data**: 600 listings across 20 cities, 20 categories, 5 sources

## Data Collection
- Data is generated by `scripts/scraper.py`, which simulates business listings from Indian cities.
- The script creates realistic names, addresses, phone numbers, categories, and sources.
- Generated data is saved to JSON and then seeded into the database on backend startup.

## Quick Start

### 1. MySQL Setup

```sql
mysql -u root -p < schema.sql
```

Or run the SQL in schema.sql manually.

### 2. Backend Setup

```bash
cd backend

# Copy and configure environment
cp .env.example .env
# Edit .env with your MySQL credentials

# Install dependencies
pip install -r requirements.txt

# Start the server (auto-seeds 600 listings on first run)
uvicorn main:app --reload --port 8000
```

> **Note**: If MySQL is unavailable, the app automatically falls back to SQLite for development.

### 3. Frontend Setup

```bash
cd frontend
npm install
npm run dev
# Open http://localhost:3000
```

### 4. Generate Fresh Data

```bash
cd scripts
python scraper.py
# Outputs 600 listings to ../data/listings.json
```

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | Health check |
| GET | `/health` | DB connection status |
| POST | `/api/listings/bulk` | **Bulk insert** scraped listings |
| POST | `/api/listings` | Insert single listing |
| GET | `/api/listings` | Paginated listing fetch (city/category/source filters) |
| GET | `/api/dashboard/summary` | Total counts (listings, cities, categories, sources) |
| GET | `/api/dashboard/city-wise` | City-wise business count |
| GET | `/api/dashboard/category-wise` | Category-wise business count |
| GET | `/api/dashboard/source-wise` | Source-wise business count |
| GET | `/api/dashboard/top-categories-by-city` | Top categories for a city |

### API Docs
Visit `http://localhost:8000/docs` for interactive Swagger UI.

## Database Schema

```sql
CREATE TABLE listing_master (
    id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    business_name VARCHAR(255)  NOT NULL,
    category      VARCHAR(100)  NOT NULL,
    city          VARCHAR(100)  NOT NULL,
    address       VARCHAR(500)  DEFAULT NULL,
    phone         VARCHAR(20)   DEFAULT NULL,
    source        VARCHAR(50)   NOT NULL,
    created_at    DATETIME      DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_city     (city),
    INDEX idx_category (category),
    INDEX idx_source   (source)
);
```

## Challenges Faced

- Ensuring the backend used the correct MySQL credentials and `.env` configuration, while also supporting a fallback to SQLite for local development.
- Handling CORS and API connectivity so React could fetch dashboard data reliably from `http://localhost:8000`.
- Making the frontend and backend start from the correct project directories, especially when using Vite and the FastAPI server together.
- Generating a complete database dump from JSON and preserving the `id` primary key values for reproducible imports.

## Dashboard Features

- **Summary cards**: Total listings, cities, categories, sources
- **City-wise horizontal bar chart**: All 20 cities ranked
- **Category-wise bar chart**: Top 10 business categories
- **Source-wise donut chart**: Proportion from each directory
- **Listings table**: Paginated view with filters
